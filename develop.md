## **阶段一：项目架构与技术选型（初始化）**

**提示词目标**：确立整体代码结构，选择最适合解析 Excel（支持样式、公式）的 Rust 库，并建立 `egui` 的基础框架。

```
角色：你是一个精通 Rust 和前端/GUI 架构的资深软件专家。
任务：我想使用 Rust 语言和 egui 框架开发一个类似 Excel 的电子表格系统。

请为我设计该项目的初始架构，要求如下：
1. 技术选型：推荐 1-2 个能够完美读取 Excel（.xlsx）文件，且能完整获取单元格字体大小、字体颜色、背景颜色、公式、原始值、数据格式、冻结窗口、合并单元格等元数据的 Rust 依赖库（例如 calamine、umya-spreadsheet 等，请对比并给出最佳选择）。
2. 数据模型设计：用 Rust 的 struct 和 enum 设计核心的电子表格数据结构。至少包含：
   - SpreadsheetData（包含多个工作表、冻结范围、合并单元格信息）
   - Sheet（矩阵数据、当前滚动位置、选中单元格坐标）
   - CellStyle（字体、大小、前景色、背景色、对齐方式、数据格式化文本）
   - Cell（包含值、公式、渲染样式、合并母格/子格状态）
3. 项目目录：提供一个干净的 cargo 项目目录结构，将数据模型（model）、Excel解析（io）、UI渲染（ui）解耦。

请先输出项目目录结构和核心数据模型的 Rust 代码。

```

## **阶段二：Excel 数据导入引擎（后端核心）**

**提示词目标**：专注于文件读取，将复杂多样的 Excel 样式和元数据精准转化为你在阶段一中定义的数据结构。

```
角色：Rust 后端数据处理专家。
任务：编写一个 Excel 导入模块，将选定的 Rust Excel 库数据转换为我们自定义的 `SpreadsheetData` 结构。

具体实现要求：

2. 解析单元格时，必须完整捕获以下属性并填入 `CellStyle`：
   - 字体名称（Font Name）、大小（Size）、加粗/倾斜（Bold/Italic）。
   - 字体颜色（ARGB/RGB 转换）和背景填充色（Pattern Fill/Solid Fill 颜色）。
   - 单元格的原始值（Value）与公式字符串（Formula，如果有）。
3. 捕获工作表级别的元数据：
   - 合并单元格范围列表（例如 [A1:C3]），并在数据模型中标记出哪些是子单元格（隐藏或被合并）。
   - 冻结行数和冻结列数（Freeze Panes）。

请给出完整的导入模块代码，注意错误处理，并确保样式转换逻辑严密。

```

## **阶段三：基于 egui 的表格高性能渲染（UI核心）**

**提示词目标**：利用 `egui` 实现网格布局，并根据单元格的样式属性进行高保真渲染，同时支持滚动和冻结窗口。

```
角色：egui 界面图形学专家。
任务：使用 egui 的自定义绘制（Painter）或 TableBuilder，实现电子表格的核心网格渲染。

具体渲染要求：
1. 基础网格：根据行高、列宽绘制网格线。支持水平和垂直滚动。
2. 样式高保真渲染：
   - 读取 `CellStyle` 中的背景色，使用 `painter.rect_filled` 填充单元格背景。
   - 读取字体大小和颜色，利用 `egui::RichText` 或自定义 `FontId` 渲染文本。
3. 冻结窗口（关键难点）：如果工作表设置了冻结前 X 行或前 Y 列，这部分行/列在滚动时必须固定在顶部和左侧，不随其余网格滚动。
4. 合并单元格渲染：如果一个单元格属于合并区域（如 A1:C3），其渲染宽度和高度应跨越对应的行列，并隐藏该区域内其他子单元格的文本和内部网格线。

请提供 `egui::App` 的 `update` 函数中处理该表格视图的核心渲染逻辑代码。

```

## **阶段四：键盘导航与内联编辑（交互核心）**

**提示词目标**：实现像 Excel 一样的键盘免鼠标无缝操作，以及双击/回车进入编辑状态。

```
角色：GUI 交互与事件响应专家。
任务：为 egui 表格添加焦点管理、键盘导航以及单元格原位编辑功能。

具体交互要求：
1. 状态管理：在应用状态中记录当前选中的单元格坐标 `selected_cell: Option<(usize, usize)>`，以及是否处于编辑模式 `is_editing: bool`。
2. 键盘导航：拦截键盘事件。当未处于编辑模式时：
   - 按 `Tab` 或 `右方向键`：选中单元格右移一列。
   - 按 `左方向键`：选中单元格左移一列。
   - 按 `上方向键` / `下方向键`：选中单元格上移/下移一行。
3. 单元格编辑：
   - 当选中单元格并按下 `Enter` 或 `F2`，或者鼠标双击时，进入编辑模式。
   - 该单元格变为 `egui::TextEdit::singleline` 组件，允许用户修改文本。
   - 在编辑模式下，按 `Enter` 提交修改并下移一行；按 `Esc` 取消修改并退出编辑。

请给出完整的事件监听与状态切换代码示例。

```

## **阶段五：顶栏名称框与公式栏（辅助 UI）**

**提示词目标**：在表格上方构建 Excel 标配的“名称框”和“公式输入框”。

```
角色：egui 布局专家。
任务：在表格上方开发一个多功能的工具栏，包含“名称框”和“公式/值显示栏”。

具体布局与联动要求：
1. 界面布局：在表格最上方，左侧为“名称框”（较窄），右侧为“公式栏”（较宽），横向排列。
2. 名称框联动：

   - 如果当前选中单元格存在公式（如 `=SUM(A1:A10)`），公式栏必须显示该公式字符串。
   - 如果没有公式，公式栏显示其当前的实际文本/数值。
   - 用户可以直接在顶部的公式栏中修改内容，修改结果实时同步回当前选中的单元格中。

请提供该工具栏的 egui UI 布局及双向数据绑定代码。
```

## **阶段六：行列右键菜单（动态操作）**

**提示词目标**：实现右键点击行列标头，动态插入或删除行列的功能。

```
角色：Rust 数据结构与 UI 交互专家。
任务：为表格的行号列（最左侧）和列字母行（最顶部）添加右键上下文菜单（Context Menu），实现行列的增删。

具体操作要求：
1. 右键检测：
   - 当鼠标右键点击最顶部的列标头（如 A, B, C...）时，弹出列操作菜单。
   - 当鼠标右键点击最左侧的行标头（如 1, 2, 3...）时，弹出行操作菜单。
2. 菜单功能项：
   - 列菜单：包含“在左侧插入列”、“在右侧插入列”、“删除当前列”。
   - 行菜单：包含“在上方插入行”、“在下方插入行”、“删除当前行”。
3. 后端数据联动：
   - 插入行列时，底层二维数据结构（如 `Vec<Vec<Cell>>`）必须动态扩容，并修正受影响的选中坐标及合并单元格索引。
   - 删除行列时，对应的内存数据释放，后续行列索引前移。

请给出如何使用 `egui::Response::context_menu` 实现该功能以及底层数据修改的逻辑代码。

```
