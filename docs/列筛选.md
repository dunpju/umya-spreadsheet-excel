升级 `E:/share/umaya-spreadsheet-excel/src/gui/widgets/search.rs` 中 `draw_search_window` 函数及 `SearchWindowState` 等相关结构体的"列筛选"功能。本项目为基于 Rust 的电子表格 GUI 应用（使用 `egui` 框架）。请先完整阅读 `search.rs`，理解现有的搜索窗口构建流程、事件处理机制、`SearchWindowState` 的全部字段及其在搜索/替换流程中的作用，然后再实施以下改动：

**1. 数据结构扩展**
- 在 `SearchWindowState` 中新增 `column_filters: Vec<ColumnFilter>` 字段（追加到结构体末尾，不得修改或重排已有字段）。
- 定义 `ColumnFilter` 结构体，包含：目标列索引（`usize`）、筛选值（`String`）。
- 定义 `enum FilterLogic { And, Or }`，在 `SearchWindowState` 中新增 `filter_logic: FilterLogic` 字段（同样追加到末尾）。

**2. 多列筛选逻辑**
- 默认采用 `And` 逻辑：仅当行数据在所有指定列中均匹配对应筛选值时，该行才被视为匹配。
- 支持 `Or` 逻辑：行数据在任意一条筛选条件对应的列中匹配即可。
- 筛选条件为空时，不产生额外过滤效果，行为与升级前一致。

**4. UI 交互改动（仅限 `draw_search_window` 函数内部）**
- 在搜索窗口的列筛选区域，每条筛选条件渲染为一行独立控件组：列选择下拉框（从配置 + 单元格数据解析出的下拉选项列表，`SearchWindowState.column_options`）、筛选值输入框（`Input`）、筛选逻辑切换按钮（AND/OR 切换控件`RadioButton`，切换时更新 `SearchWindowState.filter_logic`）、删除按钮（`Button`，至少保留第一条不可删除或提供合适的空状态处理）。
- 在`列筛选:`同行右侧新增"添加筛选条件"按钮（`Button`），点击后追加一条默认空条件并刷新 UI。
- 点击"搜索"按钮时，读取所有筛选条件及逻辑模式，在现有数据匹配流程中叠加列筛选判断（作为额外过滤条件，与关键词搜索取交集），筛选不匹配的行加入 `hidden_rows` 或等效机制。

**5. 兼容性约束（必须严格遵守）**
- 现有文本搜索、替换功能的行为和结果不得有任何变化。
