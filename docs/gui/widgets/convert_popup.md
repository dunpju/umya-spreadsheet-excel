## 模块职责与窗口行为（UI 布局）

`draw_convert_popup` 渲染一个**可拖拽调整大小的** `egui::Window`（实现方式参照预警规则配置弹窗 `draw_alert_popup`，见 [`alert_popup.md`](./alert_popup.md)）：

| 配置 | 取值 | 说明 |
|------|------|------|
| `Window::new("转换工具")` | — | 自定义标题栏（`title_bar(false)`），内部手绘「转换工具」标题 + ✖ 关闭按钮 |
| `.id(egui::Id::new("convert_popup"))` | — | 固定 id，egui 据此**持久化窗口的位置与尺寸**（用户拖拽调整后被记住） |
| `.resizable(true)` | — | **可拖拽边角/边缘调整大小**（本次改造：原为 `resizable(false)` 固定尺寸） |
| `.collapsible(false)` | — | 不允许折叠 |
| `.default_size` | `(440, 360)` | 首次默认尺寸（用户调整后以记忆尺寸为准） |
| 内容约束 | `set_min_width(440)`、`set_min_height(240)` | 最小尺寸下限，防止过度缩小导致内容不可用（原为 `min/max_width=440` 固定宽度，现**取消 max_width 上限**） |

### 自适应布局（调整窗口大小时内容正确响应）

- **多行文本输入框**：`ScrollArea::vertical().max_height(text_h)` +
  `TextEdit::multiline(desired_width(INFINITY)).desired_rows(按高度动态计算)`，
  其中 `text_h = available_height - 40`（下限 `120px`，40px 预留给底部分隔线 + 按钮行），
  `desired_rows = ceil(text_h / text_style_height(Body))`。
  两个约束同时满足：① `ScrollArea::max_height(text_h)` **硬封顶**可视高度——内容再多也只显示 `text_h`、
  超出滚动，不会把弹窗撑高；② `desired_rows` 按可用高度动态算出，让 TextEdit 固有高度 ≈ `text_h`，
  从而**填满**可视区、消除与按钮行的空白间隙。
  > 迭代历程（同一问题的三轮）：① 固定 `desired_rows(7)` → 窗口拉高后 TextEdit 仍 7 行、底部留白；
  > ② 改 `ui.add_sized` 强制铺满 → 空内容时填满，但**输入多行后 TextEdit 随内容无限增高、把弹窗撑高**
  > （`add_sized` 分配矩形但不封顶，无 `desired_rows` 的 multiline 高度由内容驱动）；③ 最终回到 `ScrollArea`
  > 封顶 + 动态 `desired_rows` 填满，兼顾「不留白」与「不撑高」。
- **进度条**：`desired_width = (available_width - 96).max(40)` —— 宽度随窗口宽度伸缩（原固定 `370`）。
- **开始转换按钮**：`right_to_left` 布局右对齐。
- **错误/成功提示**：底部 `error_message`（红）/`success_message`（绿）标签，位于按钮行下方。
- 规则预解析与 `can_convert` 判定已**移出 `ui` 闭包、提前计算**（不依赖 `ui` 借用），使布局与逻辑解耦。

### 状态字段（`ConvertPopupState`，本次改造**未改动**）

`visible`、`text`、`progress`、`error_message`、`success_message`、`is_converting` 等全部字段保持不变；
本次仅将 `draw_convert_popup` 的窗口属性与内部布局由「固定尺寸」改为「可调整 + 自适应」。

### 「开始转换」按钮启用条件（**非 Bug**）

`can_convert = !is_converting && excel_data.is_some() && file_path.is_some() && has_valid_rules`。
即按钮可点击需**同时**满足：① 已导入文件（`excel_data`/`file_path` 非 `None`）；② 输入框含至少一条
可解析规则（`parse_rules` 返回 `Ok`）；③ 非转换中。**未导入文件时按钮为灰色属正常行为**（转换必须有源文件）。
`is_converting` 当前恒为 `false`（转换同步执行），故实际门槛是「已导入文件 + 规则合法」。

## 三、转换规则语法定义

每条规则的基本结构为：
```
源范围 -> 目标起始位置 : 方向标记 ~ ;
```

### 3.1 方向标记说明

| 标记 | 含义 |
|------|------|
| `|`  | 纵向填充（向下延伸），源数据按行优先顺序逐一填入目标列的连续行 |
| `-`  | 横向填充（向右延伸），源数据按列优先顺序逐一填入目标行的连续列 |

### 3.2 源范围语法

| 语法 | 说明 | 示例 |
|------|------|------|
| `A2:M2` | 连续范围（A2 到 M2，跨列） | 见规则 1 |
| `A3:A12` | 连续范围（A3 到 A12，跨行） | 见规则 3 |
| `(N1:O1):(BV1:BW1)` | 合并单元格范围，括号内每两个单元格为一组合并单元格，从 N1、O1 开始到 BV1、BW1 结束 | 见规则 2 |
| `(N3+2):BV3` | 带步长的范围，从 N3 开始每隔 2 格取一个单元格，直到 BV3 | 见规则 5 |

### 3.3 目标位置语法

| 语法 | 说明 | 示例 |
|------|------|------|
| `A1` | 普通起始单元格 | 见规则 1、5 |
| `(B1:C1):` | 目标为合并单元格起始（B1 与 C1 两两合并），后续横向延续时每个位置也是合并单元格 | 见规则 3 |

---

## 四、规则示例（共 5 种类型）

**规则 1** — 整行转纵向单列：
```
A2:M2 -> A1 : | ~ ;
```
含义：将原表 A2~M2（一行 13 列）的数据，按列顺序纵向填入目标表从 A1 开始向下的列中。
即：A2→A1, B2→A2, C2→A3, ..., M2→A13。

**规则 2** — 合并单元格对转纵向单列：
```
(N1:O1):(BV1:BW1) -> A15 : | ~ ;
```
含义：将原表从 N1、O1 合并单元格开始，到 BV1、BW1 合并单元格为止的每一对合并单元格，纵向填入目标表从 A15 开始向下的列中。
即：N1→A15, P1→A16, R1→A17, ...

**规则 3** — 整列转横向单行（含合并单元格目标）：
```
A3:A12 -> (B1:C1): - ~ ;
```
含义：将原表 A3~A12（一列 10 行）的数据，横向填入目标表从 B1（与 C1 合并）开始向右的行中。
即：A3→B1, A4→D1, A5→F1, ...

**规则 4** — 整行转横向单行：
```
N1:BW1 -> B14 : - ~ ;
```
含义：将原表 N1~BW1 的数据，横向填入目标表从 B14 开始向右的行中。
即：N1→B14, O1→C14, P1→D14, ...

**规则 5** — 带步长跳跃取值转纵向单列：
```
(N3+2):BV3 -> B15 : | ~ ;
```
含义：从原表 N3 开始每次跳 2 格（N3→P3→R3→...→BV3），将取到的值纵向填入目标表从 B15 开始向下的列中。
即：N3→B15, P3→B16, R3→B17, ...

**规则语法：**
```
(起始列:结束列)起始行:(起始列:结束列)结束行->目标起始单元格(合并范围):目标结束单元格(合并范围):-~;
```

**具体示例：**
```

```

**含义说明：**
- 源数据范围：列 A 到列 M，每列的行范围为第 3 行到第 12 行（即 A3:A12、B3:B12、...、M3:M12，共 13 列 × 10 行的数据块）
- 转换方式：按列依次将每列的 10 行数据，横向填充到目标区域中
- 目标区域：从 B1（与 C1 合并）开始向右的行中依次填入；下一列的数据则从 B2（与 C2 合并）开始向右填入；依此类推，直到第 13 列的数据从 B13（与 C13 合并）开始向右填入
- 具体映射：A3→B1, A4→D1, A5→F1, ...（每个源值间隔一列填入，为合并单元格 C 列留空），同理 L3→B13, ..., L12→T13
- `:-~` 后缀表示按行方向扩展填充

**等价展开（该简化规则等同于以下 13 条逐列规则）：**
```
A3:A12->(B1:C1):-~;
B3:B12->(B2:C2):-~;
C3:C12->(B3:C3):-~;
D3:D12->(B4:C4):-~;
E3:E12->(B5:C5):-~;
F3:F12->(B6:C6):-~;
G3:G12->(B7:C7):-~;
H3:H12->(B8:C8):-~;
I3:I12->(B9:C9):-~;
J3:J12->(B10:C10):-~;
K3:K12->(B11:C11):-~;
L3:L12->(B12:C12):-~;
M3:M12->(B13:C13):-~;
```

**核心逻辑总结：**
该语法实现了「多列纵向数据 → 逐行横向展开」的批量转置操作，源数据中每列的行范围相同，目标区域按列索引递增分配行号，且每行目标起始列保持一致（B 列），同时通过合并单元格机制（B 列与 C 列合并）自动处理列间距。这是一种将重复的逐列转换规则简化为单条范围表达式的语法糖。
